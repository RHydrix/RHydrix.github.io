Top of file

eg. |const #variableName# = require('~Modual or File~');|

~function #functionName#(){~

Your code goes here

~return~

}

~module.exports = {#functionName#};~ This is required for wafflesbot to access it

How to Submit Your Plugin

If you dont want to use the command line, you can use github desktop on Windows and Mac which does everything I describe below in a simple and intuitive application and is alot more visual and easy

If you are a part of the Waffl3Development team, you can either use your own GitHub repository or you can use your personal Waffl3Development Team folder on the Waffl3Development Github

Make sure you create or have a specific folder for your finished plugins

Open GitBash and navigate to the folder, GitBash will allow you to open a window directly in that folder via right click

type "git init"

type "git remote add #remoteRepository#" where the remoteRepository is your Github repository or the Waffl3Development repository

*Note: if you already have things in that remote repository type "git clone #remoteRepository#" where the remoteRepository is your Github repository or the Waffl3Development repository

anything placed into that folder will be able to be sent to the github repository but you have to add, commit and push them first

type "git add #fileName#" or "git add ." to add all files, this makes it so git knows what to add to the repository

type "git commit -m "#commit message#" " Git requires you to add commit messages which tells people who are looking at the repository what was changed and when, it could also just be random letters

type "git push origin master" Finally you push all the changes you made to the repository location
